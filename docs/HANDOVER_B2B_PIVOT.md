# 📋 My Chrome Guide: B2B SaaS 피벗 프로젝트 인수인계서 (v3.5)

본 문서는 'My Chrome Guide' 프로젝트를 단순 유틸리티에서 **기업용 인터랙티브 온보딩 솔루션**으로 전환하기 위해 진행된 주요 업데이트 내역과 향후 로드맵을 기록한 문서입니다.

---

## 🗓️ 프로젝트 진행 현황 (2026-04-03 기준)

### ✅ 완료된 작업 (1~4순위)

#### 1. 노코드(No-Code) 가이드 제작 스튜디오 구축
*   **목적:** 비개발자도 웹 사이트 위에서 시각적 가이드를 제작할 수 있는 환경 제공.
*   **주요 기능:**
    *   `popup.html` 내 WYSIWYG 스텝 에디터 탑재.
    *   각 스텝별 **가이드 툴팁(Tooltip)** 작성 기능 추가.
    *   매크로 실행 시 대상 요소 하이라이트와 함께 안내 말풍선 렌더링.
*   **관련 파일:** `popup.html`, `js/actions.js`, `content_script.js`

#### 2. 요소 추적 및 AI 자가 복구(Target Repair) 고도화
*   **목적:** UI 변경 시에도 가이드가 고장 나지 않는 '치유형' 엔진 구축.
*   **주요 기능:**
    *   녹화 시 단일 식별자 외에 `tagName`, `id`, `className`, `innerText`, `placeholder` 등 **다중 메타데이터** 수집.
    *   `MacroAI` 모듈을 확장하여 로컬 AI(Gemini Nano) 및 휴리스틱 가중치 알고리즘에 메타데이터 반영.
*   **관련 파일:** `js/ai-repair.js`, `content_script.js`

#### 3. 분석 대시보드(Analytics) 및 트래킹 시스템
*   **목적:** 솔루션 도입 효과(ROI) 및 사용자의 학습 완수율 증명.
*   **주요 기능:**
    *   매크로 종료 시 성공/실패 여부를 `background.js`를 통해 로컬 스토리지에 누적 기록.
    *   팝업 내 '학습 통계' 탭 신설: 매크로별 성공률, 실행 횟수, 마지막 실행일 시각화.
*   **관련 파일:** `js/store.js`, `background.js`, `js/ui.js`

#### 4. 권한 관리(RBAC) 및 프라이빗 워크스페이스
*   **목적:** 기업 내 부서별 데이터 격리 및 관리자/사용자 역할 분리.
*   **주요 기능:**
    *   '개인용(Personal)' / '회사용(Company)' 워크스페이스 전환 로직 구현.
    *   **관리자(Admin) vs 뷰어(Viewer)** 권한 시뮬레이션: 뷰어는 생성/수정/삭제 권한 차단.
    *   보안 강화: 모든 `innerHTML` 사용을 제거하여 XSS 취약점 0건 달성 (Integrity Check 통과).
*   **관련 파일:** `js/store.js`, `js/ui.js`, `popup.js`, `scripts/check-data.js`

## 🚀 향후 과제 (로드맵)

### 5순위: 크로스 도메인(Cross-Domain) 릴레이 브라우징 지원 🚨
*   **필요성:** 실제 업무 프로세스는 여러 웹사이트(예: Jira → 사내 ERP → Slack)를 거치며 진행됨.
*   **구현 방향:**
    *   탭이 이동하거나 새 창이 열려도 `currentExecutionId`를 유지하며 가이드 세션을 이어가는 로직 필요.
    *   `background.js`를 중앙 컨트롤러로 활용하여 도메인 간 가이드 상태(Step Index)를 동기화.
    *   `content_script.js` 로드 시 백그라운드에 '현재 실행 중인 태스크가 있는지' 체크하는 핸드셰이크 강화.

### 6순위: 멀티포맷 내보내기 (Export) 📄
*   **필요성:** 확장 프로그램을 설치하지 않은 사용자나 오프라인 보고용 문서가 필요한 환경 대응.
*   **구현 방향:**
    *   제작된 인터랙티브 가이드를 **PDF 매뉴얼**로 자동 변환.
    *   **Notion 임베드**용 Markdown 형식 또는 스크린샷 기반 시퀀스 이미지 생성 기능.
    *   가이드 공유 시 필요한 권한 설정(비밀번호 등) 기능 검토.

## 🛠️ 기술적 참고 사항 (인수인계용)

1.  **데이터 무결성 검사:** 변경 사항 적용 후 반드시 `node scripts/check-data.js`를 실행하여 33개 항목의 패스 여부를 확인할 것.
2.  **보안 원칙:** UI 렌더링 시 `textContent`와 `insertAdjacentHTML`을 엄격히 구분하여 사용할 것. 사용자 입력값(Tooltip 등)이 포함된 영역은 절대로 직접 삽입하지 말 것.
3.  **상태 관리:** 모든 영구 저장은 `js/store.js`의 `storageSchema`를 통해 정의되어야 하며, `chrome.storage.local`과 State가 항상 동기화되도록 유지할 것.
